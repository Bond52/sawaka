# Contributing to Sawaka

Thank you for your interest in contributing to Sawaka!
We welcome developers, designers, testers, and content creators.

## üß© How to Contribute

### 1. Fork the Project
Click **Fork** at the top of the GitHub repository.

### 2. Create a Branch
```bash
git checkout -b feature/my-feature
```

### 3. Develop Your Contribution
- Follow coding best practices
- Add comments where necessary
- Test locally (see [README](README.md#running-tests-locally))

### 4. Clean Commit
```bash
git commit -m "feat: short description of the feature"
```

### 5. Create a Pull Request
Clearly describe what your PR does. **CI must pass** (GitHub Actions). The workflow is the Definition of Done; see [README ‚Äî CI and Definition of Done](README.md#ci-and-definition-of-done).
